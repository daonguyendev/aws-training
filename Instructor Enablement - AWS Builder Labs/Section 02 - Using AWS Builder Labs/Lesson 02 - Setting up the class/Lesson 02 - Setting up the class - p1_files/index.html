Setting up the class

The following video describes the process for setting up a class in AWS Builder Labs.

Note to ATP Training Coordinators only:

If you are a Training Coordinator for an ATP, you may be the one setting up the class environment for your ATP instructor depending on your organization's internal process. This lesson outlines the steps required to set up the environment. 

Advance to the end of this lesson for an ATP-specific walk through of the Multiple Gilmore ID Support feature that supports multi-national deliveries within one AWS Classroom.

Within the week before training delivery

  • Step 1

    Logging in

    Internal AAIs should use the Amazon Employee option to log in.

  • Step 2

    Additional login options

    Freelancer and ATP AAIs (as well as ATP Training Coordinators) should use the standard Amazon retail portal, AWS Partner Central account (if applicable), one-time passcode (OTP) to log in.

  • Step 3

    Landing Page

    Once logged in, you will be taken to the AWS Builder Labs landing page. Here you will see the option 'Create class'. Select this to get started.

  • Step 4

    Create Class web form

    Fill in all the empty fields of the 'Create Class' web form. Below are the required sections in the web form:

    • Course information
    • Class provider
    • Class access
    • Class time
    • Class location

    Once all fields are entered, select the 'Save' button.

  • Step 5

    Class Details summary page

    Once you have created the class successfully, you will be able to view the Class Details summary page. Under the following sections on the summary page, you will need to take the following actions before class:

    • Student access URL
      • Copy and share the student access URL with your students
    • Student roster
      • Select the 'Add student list' button to add students to the roster by email. The list added must be comma delimited. (Note: This field is case sensitive)

  • Step 6

    Edit details

    If a change to the class configuration is needed, you can navigate to the Edit page by selecting the 'Edit' button from the top of the Class Details page. Please note that not all fields are editable after the class has been created.

Multiple Gilmore ID Support Feature (ATP only)

This new feature allows ATPs to support multi-national deliveries within one AWS Classroom. Review the steps below for an overview of how to use the feature when setting up the class.

Introduction

The Multiple Gilmore ID Support Feature allows ATPs to now allocate Gilmore licenses aligned with the student location to better track the cost associated to each license in it's respective country.

Step 1

Logging in for Multi Region Deliveries

ATP Instructors or ATP training coordinators (TC) log into AWS Classrooms and choose a multi-national provider that will deliver the class to any child providers that have different Gilmore license IDs. 

Note: In the screenshots for this example, the ATP names are represented by names such as "Beaker Test", "Flask Test", "Dropper Test", and "Pipette Test". The actual names of the ATP by region will be displayed for users.

Step 2

Editing eKit Allocations

The ATP Instructor or TC creating the class will need to specify the number of students from each Gilmore license child provider that rolls up to the parent/multi-national provider who is delivering the class.  

Step 3

Number of Licenses Available

If there are not enough eKits purchased for that particular child provider, the ATP Instructor or TC will need to purchase more to meet the class size.

Step 4

Returning Licenses to Stock

To recoup a license from a student on the roster who was not present, reduce the overall class size prior to the end-date to avoid contacting Gilmore directly.

Licenses are reserved or released as you add or remove students to the class, and will be returned to the virtual stock within 24 hours of the class end date.
